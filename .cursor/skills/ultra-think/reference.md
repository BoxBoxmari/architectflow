# Ultra Think — Reference

Full process checklist, extended output template, and usage examples. Use when applying the full deep-analysis process or checking structure.

---

## 1. Step-by-Step Checklist

- [ ] **Initialize**: Acknowledge request; set context for systematic reasoning.
- [ ] **Parse**: Core challenge, stakeholders, constraints, implicit requirements, unknowns; question assumptions.
- [ ] **Technical**: Feasibility, scale, performance, maintainability, security, technical debt.
- [ ] **Business**: Value, ROI, time-to-market, competitive angle, risk/reward.
- [ ] **User**: Needs, pain points, usability, accessibility, edge cases, journeys.
- [ ] **System**: System-wide impact, integration, dependencies, coupling, emergent behavior.
- [ ] **Solutions**: 3–5 options; each with pros/cons, complexity, resources, risks, long-term implications.
- [ ] **Deep dive**: Implementation outline, pitfalls, mitigations, phased/MVP, second/third-order effects, failure modes.
- [ ] **Cross-domain**: Parallels, patterns, analogies, combinations.
- [ ] **Challenge**: Devil’s advocate, weaknesses, "what if", unintended consequences.
- [ ] **Synthesize**: Key factors, trade-offs, novel insights.
- [ ] **Output**: Use structured template (Problem Analysis → Options → Recommendation → Alternative Perspectives).
- [ ] **Meta**: Uncertainty, biases, confidence levels, further research or expertise.

## 2. Extended Output Template

```markdown
## Problem Analysis
- Core challenge
- Key constraints
- Critical success factors

## Solution Options
### Option 1: [Name]
- Description
- Pros/Cons
- Implementation approach
- Risk assessment

### Option 2: [Name]
[Similar structure]

### Option 3: [Name]
[Similar structure]

## Recommendation
- Recommended approach
- Rationale
- Implementation roadmap
- Success metrics
- Risk mitigation plan

## Alternative Perspectives
- Contrarian view
- Future considerations
- Areas for further research
```

## 3. Usage Examples

- **Architectural**: "Should we migrate to microservices or improve our monolith?"
- **Scaling**: "How do we scale to 10x traffic while reducing costs?"
- **Strategy**: "What technology stack for our next-gen platform?"
- **Design**: "How can we improve our API for developers while keeping backward compatibility?"

Invoke when the user asks for deep analysis, ultra-think, or complex decision support on such topics.
